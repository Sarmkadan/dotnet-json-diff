# JsonPatchFormatterTestsExtensions

The `JsonPatchFormatterTestsExtensions` class provides a set of static extension methods designed to simplify the construction and verification of JSON Patch (RFC 6902) documents in unit tests. These methods operate on `JsonPatchDocument` instances and on `JsonElement` representations of individual patch operations, enabling concise assertions about operation count, paths, values, and types without manual JSON traversal.

## API

### `ToJsonPatchWithDocument`

```csharp
public static (string Json, JsonElement Document) ToJsonPatchWithDocument(this JsonPatchDocument patch)
```

Serializes the `JsonPatchDocument` to its JSON representation and returns both the raw JSON string and a parsed `JsonElement` representing the patch array.

- **Parameters**: `patch` – The `JsonPatchDocument` to serialize.
- **Returns**: A tuple containing the JSON string and the corresponding `JsonElement` (the root of the patch array).
- **Throws**: `ArgumentNullException` if `patch` is `null`.

### `HasOperationCount`

```csharp
public static void HasOperationCount(this JsonElement document, int expectedCount)
```

Asserts that the JSON Patch document (represented as a `JsonElement` array) contains exactly the specified number of operations.

- **Parameters**:
  - `document` – A `JsonElement` that must be a JSON array of patch operations.
  - `expectedCount` – The expected number of operations.
- **Throws**: `InvalidOperationException` if `document` is not an array, or if the actual count does not match `expectedCount`.

### `GetPath`

```csharp
public static string GetPath(this JsonElement operation)
```

Extracts the `"path"` property from a single patch operation `JsonElement`.

- **Parameters**: `operation` – A `JsonElement` representing a patch operation (must contain a `"path"` property).
- **Returns**: The value of the `"path"` property as a string.
- **Throws**: `InvalidOperationException` if the `"path"` property is missing or is not a string.

### `GetValue`

```csharp
public static JsonElement GetValue(this JsonElement operation)
```

Extracts the `"value"` property from a single patch operation `JsonElement`.

- **Parameters**: `operation` – A `JsonElement` representing a patch operation (must contain a `"value"` property).
- **Returns**: The `JsonElement` of the `"value"` property.
- **Throws**: `InvalidOperationException` if the `"value"` property is missing.

### `ToOperations`

```csharp
public static IReadOnlyList<JsonElement> ToOperations(this JsonElement document)
```

Converts a JSON Patch document `JsonElement` (a JSON array) into a read-only list of individual operation `JsonElement` instances.

- **Parameters**: `document` – A `JsonElement` that must be a JSON array.
- **Returns**: An `IReadOnlyList<JsonElement>` where each element is a patch operation.
- **Throws**: `InvalidOperationException` if `document` is not an array.

### `HasValue<T>`

```csharp
public static void HasValue<T>(this JsonElement operation)
```

Asserts that the `"value"` property of a patch operation can be deserialized to the specified type `T`.

- **Parameters**:
  - `operation` – A `JsonElement` representing a patch operation with a `"value"` property.
  - `T` – The expected type of the value.
- **Throws**: `InvalidOperationException` if the `"value"` property is missing or cannot be deserialized to `T`.

### `GetFirstOperationNumericValue`

```csharp
public static int GetFirstOperationNumericValue(this JsonElement document)
```

Returns the numeric value of the `"value"` property from the first operation in the patch document.

- **Parameters**: `document` – A `JsonElement` representing a JSON Patch array (must contain at least one operation with a numeric `"value"`).
- **Returns**: The integer value of the first operation’s `"value"`.
- **Throws**: `InvalidOperationException` if the document is empty, the first operation lacks a `"value"`, or the value is not a number.

### `GetFirstOperationStringValue`

```csharp
public static string GetFirstOperationStringValue(this JsonElement document)
```

Returns the string value of the `"value"` property from the first operation in the patch document.

- **Parameters**: `document` – A `JsonElement` representing a JSON Patch array (must contain at least one operation with a string `"value"`).
- **Returns**: The string value of the first operation’s `"value"`.
- **Throws**: `InvalidOperationException` if the document is empty, the first operation lacks a `"value"`, or the value is not a string.

## Usage

### Example 1: Verifying a patch generated by a differ

```csharp
using JsonDiff;
using JsonDiff.Tests.Extensions;

[Test]
public void Patch_AddsProperty()
{
    var left = JsonDocument.Parse(@"{""name"":""Alice""}");
    var right = JsonDocument.Parse(@"{""name"":""Alice"",""age"":30}");
    var differ = new JsonDiffer();
    var patch = differ.Diff(left, right);

    var (json, doc) = patch.ToJsonPatchWithDocument();
    doc.HasOperationCount(1);

    var op = doc.ToOperations()[0];
    Assert.AreEqual("add", op.GetPath()); // note: path is "/age", not "add" – this is illustrative
    // Actually check path:
    Assert.AreEqual("/age", op.GetPath());
    op.HasValue<int>();
    Assert.AreEqual(30, op.GetValue().GetInt32());
}
```

### Example 2: Using helper methods for quick assertions

```csharp
using JsonDiff.Tests.Extensions;

[Test]
public void FirstOperation_StringValue()
{
    var patchDoc = new JsonPatchDocument();
    patchDoc.AppendAdd("/name", "Bob");

    var (_, doc) = patchDoc.ToJsonPatchWithDocument();
    doc.HasOperationCount(1);
    Assert.AreEqual("/name", doc.ToOperations()[0].GetPath());
    Assert.AreEqual("Bob", doc.GetFirstOperationStringValue());
}
```

## Notes

- All methods assume that the input `JsonElement` represents a valid JSON Patch document (a JSON array of operation objects) or a single operation object. Passing malformed JSON (e.g., a non-array document to `HasOperationCount`) will throw `InvalidOperationException`.
- `GetFirstOperationNumericValue` and `GetFirstOperationStringValue` throw if the document is empty. Use `HasOperationCount` first to guard against empty patches.
- The `HasValue<T>` method uses `System.Text.Json` deserialization; it will throw if the value is `null` and `T` is a non-nullable value type, or if the JSON value type does not match `T`.
- These extension methods are static and thread-safe as long as the input `JsonPatchDocument` or `JsonElement` instances are not mutated concurrently. In typical test scenarios (single-threaded execution), no synchronization is required.
- The `ToJsonPatchWithDocument` method serializes the patch using default `System.Text.Json` options. Custom converters or formatting settings are not applied.
